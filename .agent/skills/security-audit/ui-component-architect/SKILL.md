# UI 컴포넌트 설계 스킬 (UI Component Architect Skill)

새로운 UI 컴포넌트를 설계하고 코드를 작성할 때 다음 원칙을 따르세요:

## 🎨 설계 체크리스트
1. **반응형 (Responsive)**: 모바일(Mobile-first)에서 데스크탑까지 화면 크기 변화에 대응하는 클래스(예: Tailwind의 sm:, md: 등)가 포함되어 있는가?
2. **접근성 (Accessibility - a11y)**: `aria-label`, `alt` 속성, 키보드 네비게이션(Tab 인덱스) 등 스크린 리더 사용자를 위한 처리가 되어 있는가?
3. **상태 및 속성 (State & Props)**: 컴포넌트의 Props 인터페이스가 명확히 정의되어 있으며, 불필요한 내부 상태(Local State)를 남용하지 않았는가?
4. **재사용성 (Reusability)**: 색상이나 폰트 크기 등이 하드코딩되지 않고, 프로젝트의 전역 테마(Theme) 변수를 활용하고 있는가?

## 🛠️ 컴포넌트 작성 의사결정 트리
- **이 컴포넌트가 서버 데이터를 가져와야 하는가?**
  - **Yes**: 데이터 패칭 로직(Container)과 화면 렌더링 로직(Presentational)을 분리하여 작성하세요.
  - **No**: 순수 UI 렌더링에 집중한 함수형 컴포넌트로 작성하세요.

## 📝 결과물 제공 방법
- 컴포넌트의 전체 코드를 제공하세요.
- 복잡한 로직이나 접근성 처리를 한 부분에는 주석을 달아 이유를 설명하세요.
