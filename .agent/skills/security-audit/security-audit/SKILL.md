# 보안 감사 스킬 (Security Audit Skill)

코드의 보안을 리뷰할 때 다음 단계를 엄격히 따르세요:

## 🛡️ 리뷰 체크리스트
1. **입력값 검증 (Input Validation)**: 사용자 입력(URL 파라미터, Form 데이터 등)이 렌더링/저장되기 전에 적절히 이스케이프(Escape)되거나 살균(Sanitize) 처리되었는가? (XSS 방어)
2. **데이터 접근 (Data Access)**: 데이터베이스 쿼리나 API 요청 시, 변수가 하드코딩되지 않고 파라미터화(Parameterized) 되어있는가? (Injection 방어)
3. **인증 및 권한 (Auth & Authorization)**: 중요 데이터나 관리자 페이지 접근 시 사용자 권한을 검증하는 로직이 누락되지 않았는가?
4. **민감 정보 노출 (Sensitive Data)**: API 키, 비밀번호, 환경 변수 등이 코드나 콘솔 로그에 평문으로 노출되어 있지 않은가?

## 📝 리포트 제공 방법
- 취약점이 발견된 **정확한 파일명과 줄 번호**를 명시하세요.
- 해당 코드가 왜 위험한지(어떤 방식의 공격이 가능한지) 설명하세요.
- 취약점을 수정할 수 있는 **안전한 코드 스니펫(대안)**을 반드시 제공하세요.
- 발견된 문제가 없다면 "보안 특이사항 없음"이라고 명확히 답변하세요.
